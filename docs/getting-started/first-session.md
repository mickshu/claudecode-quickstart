# 第一次会话

## 启动

进入一个**真实项目目录**(空目录也行,但有代码会更直观):

```bash
cd ~/your-project
claude
```

你会看到一个交互式 REPL,顶部是模型与工作目录信息,底部是输入框。

::: tip 工作目录很重要
Claude 默认只能读写当前工作目录及其子目录里的文件。要让它访问目录外的内容,用 `--add-dir` flag 或在会话内通过对话让它使用绝对路径(仍受权限限制)。
:::

## 第一句 prompt

不要从"帮我重构整个项目"开始。第一句 prompt 应当让 Claude **观察**,而非**改动**:

```
列出当前目录的关键文件,简单说一下这个项目是干什么的
```

Claude 会:

1. 调用 `LS` / `Read` 等工具读取文件
2. 把结果汇总成中文回答
3. 在每次工具调用前请求你批准(权限模式见下文)

## 看懂工具调用展示

每次 Claude 决定调用工具,你会看到类似:

```
● Read(README.md)
  ⎿  Read 42 lines
```

- 圆点 = 工具调用
- 括号内 = 工具名 + 关键参数
- `⎿` 之后 = 工具返回的简短摘要

详细工具列表见 [工具与权限](/core-concepts/tools-and-permissions)。

## 权限模式(Shift+Tab 切换)

REPL 底部会显示当前权限模式:

| 模式 | 行为 | 何时用 |
| --- | --- | --- |
| **default** | 危险/修改类工具调用前询问 | 默认,日常开发 |
| **acceptEdits** | 自动批准编辑,但 shell 仍询问 | 大量小改动场景 |
| **plan** | 只读规划,不修改任何文件 | 探索/设计阶段 |
| **bypassPermissions** | 全部跳过批准 | 沙盒/CI 环境,**慎用** |

按 **Shift+Tab** 在模式间循环切换。

## 三个魔法前缀

在输入框里:

| 前缀 | 作用 | 示例 |
| --- | --- | --- |
| `!` | 直接执行 shell,把输出注入到下一次对话 | `!git status` |
| `#` | 写入项目记忆(`CLAUDE.md`)或用户记忆 | `#这个项目的测试命令是 npm run test:unit` |
| `/` | 调用斜杠命令 / Skill | `/help` `/init` `/clear` |

## 中断与重置

| 操作 | 快捷键 / 命令 |
| --- | --- |
| 中断当前回答 | `Esc` |
| 清空当前会话上下文 | `/clear` |
| 退出会话 | `Ctrl+C` 两次 / `Ctrl+D` |
| 续接最近一次会话 | 重启 CLI 时加 `--continue` |
| 选会话续接 | `claude --resume` |

## 第一次有效任务建议

跑通"观察"之后,试一个**有产出但风险低**的任务:

```
帮我写一个 README.md 的初稿:介绍项目用途、安装方式、如何运行测试
```

- Claude 读完关键文件后会给出一份 README 草稿
- 它会用 `Write` 工具创建 `README.md`,你在权限提示里确认
- 用 `!cat README.md` 立刻看效果

## 退出

`Ctrl+D` 或连按两次 `Ctrl+C`。会话默认会被持久化,可用 `claude --continue` 续接。

::: tip 不要留长会话
单次会话越长,上下文越大、价格越高、模型越容易"飘"。**任务完成就 `/clear` 或退出**,下一个任务起一个新会话。
:::
